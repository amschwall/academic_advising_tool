REFINED TDD PROMPT PLAN (UPDATED)
======================================================

GLOBAL CONVENTIONS
------------------------------------------------------
- Testing stack:
  - Backend/unit/integration: Jest
  - API: Jest + Supertest
  - Database: Postgres test DB + per-test transaction rollback
  - Frontend components: React Testing Library + Jest
  - E2E: Playwright

- AI integration:
  - All tests mock a ClaudeClient wrapper class.

- File output format:
  - Each file begins with: // file: path/to/file.ts

- Data realism:
  - Use mock/placeholder data for COLL, major requirements, catalog, prerequisites.

- TDD discipline:
  - Tests first → failing → minimal implementation → passing → refactor.

- Determinism:
  - No unseeded randomness.
  - No real network calls.
  - Each test isolated.

- Database tests:
  - Dedicated test DB.
  - Wrap each test in a transaction and roll back.

- Auth model:
  - Simulated school SSO (OAuth-style flow via Supabase Auth)

- Requirement model:
  - Students are locked to their catalog year

- Scheduler:
  - Constraint-based + preference-aware optimization

- Caching:
  - Redis-style abstraction layer (mocked in tests)

- AI behavior:
  - Invalid AI outputs are rejected (never silently corrected)

------------------------------------------------------

STANDARD PER-PHASE TEMPLATE
---------------------------
[CONTEXT]  
Explain the feature and its role in the system.

[TEST FIRST]  
Write tests only:
- Use the specified testing stack.
- Include happy path, error cases, edge cases.
- Use mock data where real data is unknown.
- Output each file with // file: ... header.
- Do NOT write implementation yet.

[STOP]  
Stop after tests. Wait for confirmation.

[IMPLEMENTATION]  
After confirmation:
- Write minimal implementation to pass tests.
- No extra features.
- Output each file with // file: ... header.

[VERIFY]  
- Show simulated test results.
- Fix failures minimally.
- Refactor while keeping tests green.
