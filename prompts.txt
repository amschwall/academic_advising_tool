TEST-DRIVEN DEVELOPMENT PROMPTING STRATEGY FOR ACADEMIC ADVISING PLATFORM
================================================================================

PHASE 1: PROJECT FOUNDATION & SETUP
================================================================================

PROMPT 1: Project Initialization
---------------------------------
Create a Next.js 14 project with TypeScript, Tailwind CSS, and the following structure:
- App router architecture
- PostgreSQL database with Prisma ORM
- Supabase for authentication
- Zustand for state management
- Initial folder structure: /app, /components, /lib, /types, /tests

Write setup tests that verify:
- Next.js dev server starts successfully
- Database connection is established
- Environment variables are loaded correctly


PHASE 2: DATA LAYER (TDD FROM BOTTOM UP)
================================================================================

PROMPT 2: Database Schema & Tests
----------------------------------
Write Prisma schema for:
- Courses (code, title, credits, description, department, coll_attributes)
- Prerequisites (course relationships)
- Students (major, minor, completed_courses)
- Schedules (student plans)

First, write integration tests for:
- Creating/reading course records
- Querying courses by department/COLL attributes
- Prerequisite chain validation
- Student course completion tracking

Then implement the schema and CRUD operations to pass these tests.


PROMPT 3: Web Scraper with Tests
---------------------------------
Build a Playwright scraper for W&M Banner course schedules.

Test requirements:
- Mock HTML fixtures from Banner pages
- Unit tests for parsing course codes, titles, times, professors
- Integration test that scrapes and stores 1 semester of data
- Error handling for missing/malformed data

Implement scraper to pass all tests. Target: Computer Science department first.


PHASE 3: BUSINESS LOGIC LAYER
================================================================================

PROMPT 4: Requirement Validation Engine
----------------------------------------
Create a rule-based validator for degree requirements.

Write tests first:
- Test COLL requirement checking (e.g., student needs 2 more COLL 200 courses)
- Test major requirement validation (CS major needs CSCI 141, 241, 303...)
- Test prerequisite chains (can't take CSCI 303 without CSCI 241)
- Test credit hour limits (max 18 credits/semester)
- Test time conflict detection

Then implement the validation engine to pass these tests.


PROMPT 5: Schedule Generation Algorithm
----------------------------------------
Build a 4-year schedule generator.

Test cases:
- Generate valid 8-semester plan for CS major with 0 completed courses
- Respect prerequisites (no taking courses before prereqs)
- Distribute COLL requirements across semesters
- Handle course availability (only offered fall/spring)
- Reject invalid plans (return errors, not partial schedules)

Implement algorithm using constraint satisfaction approach.


PHASE 4: AI INTEGRATION LAYER
================================================================================

PROMPT 6: Claude API Integration with Mocks
--------------------------------------------
Set up Anthropic Claude API integration for course recommendations.

Write tests with mocked API responses:
- Mock successful recommendation responses
- Test prompt construction for user interests
- Test parsing of Claude's JSON responses
- Test error handling (rate limits, timeouts)
- Test that recommendations are validated against actual course catalog

Implement Claude integration to pass tests.


PROMPT 7: AI Validation Layer
------------------------------
Create a wrapper that validates all AI outputs.

Test that:
- AI-recommended courses exist in database
- AI-generated schedules meet all requirements
- AI suggestions respect prerequisites
- Hallucinated courses are caught and filtered out
- Invalid schedules are rejected with helpful error messages

Implement validation wrapper around Claude API.

GET /api/student/progress

- Test missing data handling

Implement endpoints to satisfy the test contracts.

KEY TDD PRINCIPLES TO EMPHASIZE IN PROMPTS
================================================================================

1. RED-GREEN-REFACTOR: Always write failing tests first
2. ONE TEST AT A TIME: Don't move on until current test passes
3. MINIMAL IMPLEMENTATION: Write just enough code to pass
4. TEST ISOLATION: Each test should be independent
5. MEANINGFUL ASSERTIONS: Test behavior, not implementation
6. MOCK EXTERNAL DEPENDENCIES: Use fixtures for Banner HTML, Claude API


EXAMPLE PROMPT FORMAT
================================================================================

[CONTEXT]: We're building [feature] for the academic advising platform.

[TEST FIRST]: Write comprehensive tests for:
- [specific scenario 1]
- [specific scenario 2]
- [edge case]

[REQUIREMENTS]: Tests must:
- Use [testing framework]
- Cover happy path + error cases
- Be deterministic and isolated

[IMPLEMENTATION]: Only after tests are written and failing, implement the 
minimum code to make them pass.

[VERIFY]: Run tests, show results, refactor if needed while keeping tests green.


CONCLUSION
================================================================================

This approach ensures every feature is validated, prevents regressions, and 
creates a robust, maintainable codebase from day one. Each prompt builds on 
the previous one, creating a solid foundation before moving to the next layer.
